Center-Crop to Square

geometry
–
Crop Math
$$s = \min(H,\,W)$$ $$x_0 = \left\lfloor\frac{W - s}{2}\right\rfloor,\quad y_0 = \left\lfloor\frac{H - s}{2}\right\rfloor$$ $$I_{\text{crop}} = I[y_0 : y_0+s,\; x_0 : x_0+s]$$
Why? Neural networks expect a fixed aspect ratio (1:1). Cropping from the centre keeps the subject visible without distortion.

Resize to 224 × 224

geometry
224 × 224 px
Bilinear Interpolation (per channel)
$$\text{scale} = \frac{224}{s}$$ $$I_{\text{resize}}[i,j] = I_{\text{crop}}\!\left[\frac{i}{\text{scale}},\,\frac{j}{\text{scale}}\right]$$
For a point $(x,y)$ between four neighbours $Q_{11},Q_{12},Q_{21},Q_{22}$: $$P = \frac{1}{(x_2-x_1)(y_2-y_1)}\bigl[(x_2-x)(y_2-y)\,Q_{11} + (x-x_1)(y_2-y)\,Q_{21}$$ $$\quad+(x_2-x)(y-y_1)\,Q_{12} + (x-x_1)(y-y_1)\,Q_{22}\bigr]$$
MobileNet was pre-trained on 224 × 224 images, so every input must match this exact size.

Normalise to [−1, 1]

math
Pixel intensity histogram (before vs after normalisation)
Normalisation Formula
$$\hat{x} = \frac{x}{127} - 1, \quad x \in [0,\,255]$$
$$\hat{x}_{\min} = \frac{0}{127}-1 = -1 \qquad \hat{x}_{\max} = \frac{255}{127}-1 \approx +1$$
Why normalise? Gradient descent converges much faster when all features share the same scale. Keeping values in $[-1, 1]$ prevents exploding/vanishing gradients.

$$\mathbf{T}_{\text{batch}} = \mathbf{T}[224,224,3]\;\xrightarrow{\text{expandDims}}\;\mathbf{T}_{\text{batch}}[1,224,224,3]$$
Neural networks process data in batches. The outer dimension $B$ indicates how many images are in the batch. During training $B > 1$; for single inference $B = 1$.

MobileNet Feature Extraction → 1-D Vector

deep learning
Global Average Pooling
$$\mathbf{F}_{c} = \frac{1}{H' W'}\sum_{i=1}^{H'}\sum_{j=1}^{W'} A_{i,j,c} \qquad c = 1,\ldots,C$$
MobileNetV2: $A \in \mathbb{R}^{7 \times 7 \times 1280} \;\xrightarrow{\text{GAP}}\; \mathbf{F} \in \mathbb{R}^{1280}$

Training — Loss & Gradient Descent

training
Cross-Entropy Loss per epoch
Accuracy per epoch
Cross-Entropy Loss (Softmax output)
$$\mathcal{L} = -\sum_{k=1}^{K} y_k \log\bigl(\hat{p}_k\bigr)$$ $$\hat{p}_k = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}} \qquad \text{(Softmax)}$$
Gradient Descent Weight Update
$$\theta \;\leftarrow\; \theta - \eta\,\nabla_\theta \mathcal{L}$$ where $\eta$ is the learning rate and $\nabla_\theta \mathcal{L}$ is the gradient of the loss w.r.t. weights $\theta$.

Class Comparison — 2-D Embedding View

visualisation
Simulated 2-D PCA projection of feature vectors (different colours = different classes)
Softmax Classification
Given feature vector $\mathbf{F}$, the head computes: $$z_k = \mathbf{w}_k^T \mathbf{F} + b_k$$ $$\hat{p}_k = \frac{e^{z_k}}{\sum_j e^{z_j}}$$ Predicted class: $\hat{y} = \arg\max_k \hat{p}_k$
Each dot represents one training image projected into 2-D space. The model learns a decision boundary (dashed line) that separates the classes.
